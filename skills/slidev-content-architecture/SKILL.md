# Slidev Content Architecture

## Goal

Turn rough material into a clear slide narrative.

## Workflow

1. Identify the presentation objective:
   - what the audience must remember
   - what the audience must believe
   - what the presenter will demonstrate or prove
2. Define the core storyline:
   - context
   - problem
   - model or approach
   - proof/demo
   - synthesis
   - next steps
3. Limit each slide to one main idea.
4. For each slide, define:
   - title
   - main message
   - supporting points
   - visual role
   - transition to the next slide
5. Remove slides that only repeat earlier ideas.
6. Keep slides self-supporting without turning them into documentation.
7. Add speaker notes only when they clarify flow or timing.

## Output Shape

Prefer a slide outline before writing full content:

```text
Slide N - Title
Main message:
Visual:
Key points:
Transition:
```
