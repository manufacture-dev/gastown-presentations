# Presentation Iteration Workflow

## Goal

Keep presentation iteration controlled, visible, and reversible.

## Workflow

1. Separate iteration types:
   - content
   - theme
   - layout
   - assets
   - demo flow
2. Change one category at a time when possible.
3. For visual work:
   - create a variant
   - run the deck
   - capture screenshots
   - ask for targeted feedback
4. Keep useful changes and remove failed variants.
5. Avoid mixing experimental theme work with content changes in the same commit.
6. When a pattern works, apply it consistently across related slides.
7. Keep a short summary of decisions made during iteration.

## Feedback Loop

Use this loop:

```text
change -> build -> screenshot -> feedback -> adjust -> verify
```

If repeated manual adjustments are needed, refactor the layout rather than continuing to tune individual values.
